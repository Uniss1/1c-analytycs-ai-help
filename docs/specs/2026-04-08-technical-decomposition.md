## 2. User Journey

```
ЭТАП 1: ПЕРВЫЙ КОНТАКТ
═══════════════════════
Пользователь открывает 1С Аналитику → видит дашборд "Продажи"
→ замечает синюю кнопку 💬 в правом нижнем углу
→ нажимает → открывается чат-панель
→ видит placeholder "Задайте вопрос..."

ЭТАП 2: ПЕРВЫЙ ВОПРОС ПО ДАННЫМ
════════════════════════════════
Пользователь: "Какая выручка за март?"
→ появляется индикатор "Ищу в данных..."
→ виджет отправил dashboard_context: {url: "/analytics/sales", title: "Продажи"}
→ через 2-4 сек ответ: "Выручка за март 2025: 8.2 млн ₽"
→ пользователь понимает — бот знает контекст дашборда

ЭТАП 3: УТОЧНЯЮЩИЙ ВОПРОС
══════════════════════════
Пользователь: "А в разрезе подразделений?"
→ бот помнит предыдущий вопрос (chat history)
→ "Выручка за март 2025 по подразделениям:
   | Подразделение | Сумма      |
   | Москва        | 4.1 млн ₽  |
   | СПб           | 2.8 млн ₽  |
   | Казань        | 1.3 млн ₽  |"

ЭТАП 4: ВОПРОС ПО МЕТОДОЛОГИИ
══════════════════════════════
Пользователь: "Как считается маржинальность на этом дашборде?"
→ индикатор "Ищу в базе знаний..."
→ бот ищет в Wiki.js
→ "Маржинальность рассчитывается как (Выручка - Себестоимость) / Выручка × 100%.
   Источник: статья 'Методология расчёта показателей' в базе знаний."

ЭТАП 5: ДРУГОЙ ДАШБОРД
═══════════════════════
Пользователь переходит на дашборд "Затраты"
→ открывает виджет → "Топ-5 статей затрат за Q1"
→ dashboard_context обновился автоматически
→ бот ищет по витринам дашборда "Затраты", не "Продажи"

ЭТАП 6: STANDALONE (без 1С Аналитики)
══════════════════════════════════════
Пользователь открывает /assistant/ в браузере напрямую
→ полноценный чат без привязки к конкретному дашборду
→ бот ищет по всем витринам
→ работает как общий помощник по данным + знаниям
```

## 4. Схема базы данных

### metadata.db

```sql
-- Дашборды из 1С Аналитики
CREATE TABLE dashboards (
    id          INTEGER PRIMARY KEY,
    slug        TEXT NOT NULL UNIQUE,     -- "sales", "costs", "hr"
    title       TEXT NOT NULL,            -- "Продажи"
    url_pattern TEXT NOT NULL,            -- "/analytics/sales*"
    updated_at  TEXT NOT NULL             -- ISO 8601
);

-- Витринные регистры
CREATE TABLE registers (
    id          INTEGER PRIMARY KEY,
    name        TEXT NOT NULL UNIQUE,     -- "РегистрНакопления.ВитринаВыручка"
    description TEXT NOT NULL,            -- "Выручка по подразделениям и номенклатуре"
    register_type TEXT NOT NULL,          -- "accumulation_turnover" | "accumulation_balance" | "info"
    updated_at  TEXT NOT NULL
);

-- Связь дашборд → регистр (M:N)
CREATE TABLE dashboard_registers (
    dashboard_id INTEGER NOT NULL REFERENCES dashboards(id),
    register_id  INTEGER NOT NULL REFERENCES registers(id),
    widget_title TEXT,                    -- "Выручка по месяцам" (название виджета)
    PRIMARY KEY (dashboard_id, register_id)
);

-- Измерения регистра
CREATE TABLE dimensions (
    id          INTEGER PRIMARY KEY,
    register_id INTEGER NOT NULL REFERENCES registers(id),
    name        TEXT NOT NULL,            -- "Подразделение"
    data_type   TEXT NOT NULL,            -- "Справочник.Подразделения" | "Дата" | "Строка"
    description TEXT                      -- "Подразделение организации"
);

-- Ресурсы регистра (числовые показатели)
CREATE TABLE resources (
    id          INTEGER PRIMARY KEY,
    register_id INTEGER NOT NULL REFERENCES registers(id),
    name        TEXT NOT NULL,            -- "Сумма"
    data_type   TEXT NOT NULL DEFAULT 'Число', -- "Число" | "Количество"
    description TEXT                      -- "Сумма в рублях"
);

-- Ключевые слова для поиска регистра по вопросу
CREATE TABLE keywords (
    id          INTEGER PRIMARY KEY,
    register_id INTEGER NOT NULL REFERENCES registers(id),
    keyword     TEXT NOT NULL             -- "выручка", "продажи", "доход", "оборот"
);

CREATE INDEX idx_keywords_word ON keywords(keyword);
CREATE INDEX idx_dashboard_slug ON dashboards(slug);

-- Пример данных:
-- dashboards: (1, 'sales', 'Продажи', '/analytics/sales*')
-- registers:  (1, 'РегистрНакопления.ВитринаВыручка', 'Выручка...', 'accumulation_turnover')
-- dashboard_registers: (1, 1, 'Выручка по месяцам')
-- dimensions: (1, 1, 'Период', 'Дата'), (2, 1, 'Подразделение', 'Справочник.Подразделения')
-- resources:  (1, 1, 'Сумма', 'Число'), (2, 1, 'Количество', 'Число')
-- keywords:   (1, 1, 'выручка'), (2, 1, 'продажи'), (3, 1, 'доход')
```

### history.db

```sql
-- Сессии чата
CREATE TABLE sessions (
    id          TEXT PRIMARY KEY,          -- UUID
    user_id     TEXT,                      -- из auth (nullable для anonymous)
    created_at  TEXT NOT NULL,
    last_active TEXT NOT NULL
);

-- Сообщения
CREATE TABLE messages (
    id          INTEGER PRIMARY KEY,
    session_id  TEXT NOT NULL REFERENCES sessions(id),
    role        TEXT NOT NULL,             -- "user" | "assistant"
    content     TEXT NOT NULL,
    intent      TEXT,                      -- "data" | "knowledge" | NULL
    register    TEXT,                      -- какой регистр использован (для data)
    query_text  TEXT,                      -- сгенерированный запрос 1С (для data)
    latency_ms  INTEGER,                  -- время обработки
    created_at  TEXT NOT NULL
);

CREATE INDEX idx_messages_session ON messages(session_id, created_at);

-- Кэш ответов
CREATE TABLE response_cache (
    id              INTEGER PRIMARY KEY,
    question_hash   TEXT NOT NULL UNIQUE,  -- SHA256(normalized_question + dashboard_slug)
    question        TEXT NOT NULL,
    answer          TEXT NOT NULL,
    intent          TEXT NOT NULL,
    dashboard_slug  TEXT,
    created_at      TEXT NOT NULL,
    expires_at      TEXT NOT NULL          -- TTL, например +1 hour
);

CREATE INDEX idx_cache_hash ON response_cache(question_hash);
CREATE INDEX idx_cache_expires ON response_cache(expires_at);
```

## 5. Системные промпты

### 5.1 Router (GPU 0) — классификация intent

```
Ты классификатор запросов к системе аналитики 1С. Определи тип запроса.

Ответь ОДНИМ словом — data или knowledge.

data — вопрос про конкретные цифры, показатели, метрики:
- суммы, количества, проценты за период
- сравнение периодов ("больше/меньше чем")
- топ-N, рейтинги, ранжирование
- фильтрация ("по подразделениям", "только Москва")

knowledge — вопрос про методологию, определения, инструкции:
- как считается показатель
- что означает термин
- как пользоваться дашбордом
- почему данные расходятся
- регламент, процедура, политика

Примеры:
"Выручка за март" → data
"Топ-5 подразделений" → data
"Сколько приняли в Q1" → data
"Как считается маржа?" → knowledge
"Что такое EBITDA?" → knowledge
"Почему цифры не совпадают с Excel?" → knowledge
"Какой регламент закрытия периода?" → knowledge
```

### 5.2 Query Generator (GPU 1) — генерация запроса 1С

```
Ты генератор запросов на встроенном языке запросов 1С:Предприятие. Генерируй ТОЛЬКО текст запроса, без пояснений.

СИНТАКСИС:
ВЫБРАТЬ [ПЕРВЫЕ N] поля ИЗ источник [ГДЕ условия] [СГРУППИРОВАТЬ ПО поля] [УПОРЯДОЧИТЬ ПО поля]

АГРЕГАТНЫЕ ФУНКЦИИ: СУММА(), КОЛИЧЕСТВО(), МАКСИМУМ(), МИНИМУМ(), СРЕДНЕЕ()
УСЛОВИЯ: =, <>, >, <, >=, <=, МЕЖДУ ... И ..., В (...), ПОДОБНО
СОРТИРОВКА: ВОЗР (ASC), УБЫВ (DESC)

ВИРТУАЛЬНЫЕ ТАБЛИЦЫ РЕГИСТРОВ НАКОПЛЕНИЯ:
- Обороты: РегистрНакопления.XXX.Обороты(&Начало, &Конец, , , )
  Доступны все измерения + ресурсы как суммы оборотов
- Остатки: РегистрНакопления.XXX.Остатки(&Дата, )
  Доступны все измерения + ресурсы как остатки

ПАРАМЕТРЫ: используй &Имя для параметров дат и значений.

ПРАВИЛА:
1. Всегда добавляй ПЕРВЫЕ 1000 если пользователь не указал другой лимит
2. Используй только тот регистр, который указан в метаданных ниже
3. Используй только те измерения и ресурсы, которые перечислены в метаданных
4. Для дат используй параметры &Начало, &Конец, &Дата

МЕТАДАННЫЕ РЕГИСТРА:
{metadata}

ВОПРОС: {question}
```
